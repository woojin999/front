document 영역에서 사용되어지는 이벤트는 모두 사전 정의 되어 있음

마우스, 키보드, 실행 등 이벤트의 종류가 매우 많기 때문에 웹에서 자주 사용하는 이벤트를 우선적으로 학습하길 권장

  • 마우스 이벤트 : click(), focus(), blur(), input(), change(), mouseover(), mouseout(), select(), scroll(), load(), drag() => drop() 과정, contextMenu() , event.which => mousedown(),
  • 키보드 이벤트 : event.key => keyDown(), event.keyCode / event.which => keyPress()
  • 기타 이벤트관련 메서드 : preventDefault(), target(), submit(), reset(), search()
  • 메서드를 태그 속성이나 엘리먼트에 직접 부여하는 방식이외 addEventListener('eventName', callback function) 메서드를 사용하여 구현하는 것을 권장함
  • addEventListener를 권장하는 이유 : 같은 엘리먼트에 여러 이벤트를 중복해서 부여할수 있고, 적용된 이벤트를 선택적으로 제거할 수 있다.
  • 이벤트 발생 방식 addEventListener('eventName', callback, useCapture)
    1. 캡쳐링(true) : 대상 객체에 이벤트를 발생시키기 위해 감싸고 있는 외부 엘리먼트부터 접근하여 실행하는 방식 : 외부 => 내부 접근
    2. 버블링(Default=false) : 내부에서 외부로 영향력 전달 방식
