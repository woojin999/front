예상되는 에러를 try {code...} 영역에 배치하고 만약 에러가 발생하면 catch(err){code...} 영역에서 처리하는 예외처리 프로세스

  • try 영역 : 에러발생이 예상되는 코드를 실행하는 곳
  • catch(err) 영역 : 에러발생시 예외처리 코드를 실행하는 곳
  • try ~ catch로 예외처리를 하면 프로세스를 에러로 인해 멈추지 않고 지속시킬수 있다
  • finally 영역 : 예외처리 유무에 관계없이 무조건 코드를 실행하는 영역
  • throw : 예외를 고의적으로 발생시킬 수 있는 키워드, throw + 예외발생후 실행할 코드
